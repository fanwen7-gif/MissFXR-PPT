Part 01
核心工具
我70%的AI使用量，集中在这两个工具上
TOOL 01 / 08
Claude
claude.ai · Anthropic · 网页端
我的判断：Claude现在是御三家里独一档的存在。
御三家——Claude、GPT、Gemini——能力上断崖领先其他所有产品。
而在御三家里面，Claude现在又是单独一档。

我用它做所有需要真正思考的事情。
产品方案、内容脚本、用研分析、复杂决策。

我的建议：不要把它当搜索引擎，要把它当你的万能导师请教。
具体用哪个模型
1. Opus 4.6 负责处理复杂任务，它非常擅长编程和规划
2. 简单任务用 Sonnet，它非常擅长写作和讨论
使用准备
网络环境需要**学上网
账号claude.ai 注册，支持 Google 账号登录
充值一开始完全不必，免费额度先用起来
TOOL 02 / 08
Claude Code
CLI · Anthropic · 桌面端（命令行）
和Claude是完全不同的两个东西。
Claude是对话，Claude Code是Agent——
它不只是回答你，它会自己制定计划、调用工具、执行操作、反复迭代，直到完成任务。
本质上是一个自动化执行工具。

核心场景：Vibe Coding（描述想要什么，它直接写代码并写入文档，我不需要懂代码）、
自动化创作（读取本地知识库，帮我做信息处理）。
使用准备
网络环境需要**学上网
购买API在 Anthropic 官网充值 API 额度，或购买国产 API
安装部署命令行安装，有一定门槛，跟着官方文档走是可以做到的

TOOL 03 / 08
Gemini Flash
gemini.google.com · Google · 网页端
生图工具我用过很多。我的判断：
Gemini Flash 是目前第一、没有之一的最好生图模型。

又快、又准确、又好看。
最关键的是风格极其可控——你描述什么，它给你什么，审美在线。
使用准备
去 gemini.google.com，登录 Google 账号就能用，有免费额度，足足够用。
TOOL 04 / 08
即梦
Dreamina · ByteDance · 网页端 + 手机端
但我有时候也会用即梦——什么时候？

有中文需求的时候。

Gemini 对中文的控制力还不够好，
图里要出现中文字、或者需要中文场景的画面感，即梦处理得更准确。
使用准备
下载：直接搜「即梦AI」，抖音系产品，国内直接用。
TOOL 05 / 08
Gemini 3.1 Pro
gemini.google.com · Google · 网页端
Gemini 除了生图，我还用它的 3.1 Pro 模型做有审美要求的设计场景——
Web PPT 的排版设计、前端产品页面的设计。

它在视觉设计判断上有独特优势，
是 Claude 在这个细分场景下的补充。
使用准备
同一个产品，在 gemini.google.com 切换模型即可。
TOOL 06 / 08
龙虾
OpenClaw · openclaw.ai · 网页端 + 桌面端
这个工具我用它做三件事：
抓取和爬取数据、设置信息提醒、内容沉淀。

做研究、追踪行业动态的时候非常好用。
使用准备
官网注册，有中文界面。

TOOL 07 / 08
OpenAI Codex
OpenAI · 桌面端
OpenAI 出的桌面端工具，我偶尔用它做简单的数据分析。

有轻量级数据处理需求的可以试试。

TOOL 08 / 08
飞书妙记
Lark Minutes · 字节跳动 · 手机端 + 桌面端
语音转写 yyds，
我几乎录音身边所有有信息量的线下场景。
使用准备
飞书 App 内置功能，国内直接用。
My Advice
01
直接用最 SOTA 的，不需要自己去发现，
抄 best practice 就行。
02
对模型和工具要有敏感度。
Claude 是 Claude，AI 是 AI，
差距就像自行车和汽车——都是车，但不是一回事。
